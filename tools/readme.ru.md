## Доступные команды

- `osm up`: Создать и запустить контейнер с OSM Tile Server. Сервер предварительно должен быть инициирован командой `osm import`, чтобы у сервера были данные для рендеринга.

- `osm down`: Остановить и удалить контейнер OSM Tile Server.

- `osm import [имя-региона]`: Импортировать данные OSM. Если имя региона указано, его данные сначала будут загружены с geofabrik.de. Если имя региона не указано, выполняется импорт загруженного ранее региона (из `/mnt/data/region.osm.pbf`). Примеры имён регионов: `europe/belarus`, `russia/kaliningrad-oblast`, `russia`. Эта команда полностью удаляет существующую базу данных PostgreSQL и кэш тайлов Apache перед импортом.

- `osm get <имя-региона>`: Загрузить latest.osm.pbf и .poly для указанного региона с geofabrik.de и сохранить их в файлы `/mnt/data/region.osm.pbf` и `/mnt/data/region.poly`, соответственно.

- `osm logs`: Показать логи запущенного OSM Tile Server в режиме реального времени (follow).

- `osm render <lon,lat,LON,LAT,z,Z> [n]`: Запустить рендеринг тайлов для заданной географической области (bbox), минимального и максимального уровней масштабирования (z, Z) и количества потоков (n). Bbox задается в географических координатах WGS84: lon, lat - левый нижний угол, LON, LAT - правый верхний угол. Нумерация зумов начинается с 1.

- `osm pg <подкоманда>`: Управление PostgreSQL в контейнере OSM Tile Server.
  - `run`: Создать и запустить контейнер с сервером PostgreSQL.
  - `stop`: Остановить запущенный контейнер с сервером PostgreSQL.
  - `analyze`: Выполнить команду `ANALYZE` в базе данных.
  - `vacuum`: Выполнить команду `VACUUM` в базе данных.
  - `convert`: Выполнить скрипт `convert-names.sql` для смены языка карты.
